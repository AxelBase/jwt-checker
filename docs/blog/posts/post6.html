JWT Misconfigurations

Common JWT Misconfigurations You’re Probably Making

Published: November 8, 2025

JWTs are secure by design — but only if configured correctly. Here are 7 mistakes we see in production every week.

1. Using Weak Secrets

secret: "123456" → cracked in seconds.

Fix: Use 256-bit random keys. Generate with:
openssl rand -base64 32

2. No Algorithm Validation

Attacker changes alg: "none" → bypasses signature.

Fix: Explicitly allow only HS256 or RS256.

3. Long-Lived Tokens

exp: now + 30 days → stolen token = 30 days of access.

Fix: Max 1 hour. Use refresh tokens.

4. Missing iat Check

Attacker reuses old token after password reset.

Fix: Reject tokens older than last login.

5. Storing JWTs in localStorage

XSS → token theft.

Fix: Use httpOnly cookies for auth tokens.

6. No Audience/Scope

Token for API A used on API B.

Fix: Validate aud and scope.

7. Ignoring Clock Skew

Token rejected 2 seconds after exp.

Fix: Allow ±30 seconds tolerance.
